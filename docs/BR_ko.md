# 비즈니스 요구사항 정의서

**프로젝트명:** Cheongung Sim  
**작성일:** 2026-04-04 | **최종 수정:** 2026-04-12  
**버전:** 1.0 (완성)

---

## 1. 배경

2026년 미국-이란 전쟁에서 UAE에 배치된 천궁-II가 실전 투입되어 탄도미사일 96%, 순항미사일 100% 요격률을 기록했다. 이는 한국산 방공 무기체계의 실전 가치를 입증한 사례로, 향후 복합 위협 환경에서의 방공 시뮬레이션 연구의 필요성이 높아지고 있다.

본 프로젝트는 천궁-II의 실제 교전 로직을 참고하여, 복합 위협(탄도미사일, 순항미사일, 드론) 환경에서의 요격 시뮬레이션을 구현하는 것을 목표로 한다.

---

## 2. 목적

- 천궁-II의 3단계 유도 방식(INS → 지령유도 → 능동 레이더 호밍)을 시뮬레이션으로 구현
- 복합 위협 환경에서 위협 우선순위 평가 및 다중 표적 동시 교전 로직 구현
- 요격 성공률, 잔여 미사일 수 등 방어 효율 메트릭 도출

---

## 3. 범위

**Phase 1 (본 문서 기준)**
- 2D 사이드뷰 시뮬레이션 (x: 수평거리, y: 고도)
- 위협 종류: 탄도미사일, 순항미사일, 드론 (혼합)
- 아군 포대: 천궁-II 1개 포대 (화면 하단 고정)
- 방어 대상: 핵심 시설 1개 (예: 공군기지, 포대 옆 지상 배치)

**Phase 2 이후 (미정)**
- 다중 포대 협력 교전
- 3D 시뮬레이션 확장
- 강화학습 기반 자동 교전 판단

---

## 4. 주요 기능 요구사항

### 4.1 위협 생성
- 탄도미사일: 고속(마하 5 이하), 고고도, 포물선 궤적 (중력 적용)
- 순항미사일: 중속, 저고도, 수평 직선 비행 (고도 일정 유지)
- 드론: 저속, 초저고도, 불규칙 기동 (x/y 방향 모두 랜덤 변화)
- 다수의 위협이 동시 또는 순차적으로 발사 가능

### 4.2 레이더 탐지
- 탐지 반경 내 위협 감지
- 탐지 반경 외 위협은 교전 불가
- 위협 종류별 탐지 난이도 차등 적용 (드론 탐지 어려움)

### 4.3 위협 우선순위 평가 (교전통제)
- 평가 기준: 속도, 고도, 방어 대상까지 남은 거리
- 점수화하여 높은 순으로 요격 미사일 할당
- 요격 가능 시간창(Engagement Window) 내 표적만 교전

### 4.4 요격 미사일 유도 (3단계)
- **초기 (INS):** 발사 후 자체 관성항법으로 비행
- **중간 (지령유도):** 레이더가 미사일과 표적 동시 추적, 유도 명령 전송
- **종말 (능동 레이더 호밍):** 미사일 자체 레이더로 표적 추적 및 직격

### 4.5 요격 판정
- Hit-to-Kill 방식 (직접 충돌)
- 거리 임계값 이하 진입 시 요격 성공 판정
- 요격 실패 시 위협이 방어 대상에 도달

### 4.6 메트릭 수집
- 요격 성공률 (위협 종류별)
- 소모된 요격 미사일 수
- 방어 대상 피격 횟수
- 교전 시간

### 4.7 시각화
- **pygame:** 실시간 2D 시뮬레이션 (위협 궤적, 요격 미사일 경로, 충돌 이벤트)
- 격자 배경, 레이더 탐지 범위 원, 위협별 trail 잔상, 폭발 이펙트
- 군사 HUD (탄약 잔량, 재장전 타이머, 위협 종류별 카운트, 기지 HP 바)
- 인트로 화면 (작전 배경 텍스트 + 스페이스바 시작)
- **matplotlib:** 시뮬레이션 종료 후 메트릭 그래프 출력 (dark theme)

### 4.8 재장전 시스템
- 초기 요격 미사일: 20발
- 10초마다 4발 자동 재장전 (최대 20발 유지)
- 잔량 5발 이하 시 "AMMO LOW" 경고 표시

### 4.9 위협 스폰 시스템
- 3방향 스폰: 오른쪽 / 위 / 아래
- 4초마다 위협 1개 개별 스폰
- 15초마다 포화공격: 위협 5~8개 동시 스폰
- 60초 전투 후 자동 종료 또는 기지 HP 0 시 종료
