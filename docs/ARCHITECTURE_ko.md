## 컴포넌트별 설명

### ThreatGenerator
위협 객체를 생성하고 매 프레임 궤적을 업데이트한다.

- **탄도미사일:** 포물선 궤적 (중력 9.8 적용), 고속(마하 5 이하), 고고도 — x 이동 + vy 감소
- **순항미사일:** 수평 직선 비행 (고도 고정), 중속, 저고도 — x만 이동
- **드론:** 불규칙 기동, 저속, 초저고도 — x/y 모두 랜덤 변화
- 시나리오 파일(JSON)에서 위협 발사 시각 및 수량 로드

### Radar
탐지 반경 내 위협을 감지하여 EngagementControl에 전달한다.

- 탐지 반경: 설정값 기반 (천궁-II 참고)
- 위협 종류별 탐지 확률 차등 적용
  - 탄도미사일: 높음
  - 순항미사일: 중간
  - 드론: 낮음 (소형, 저고도)

### EngagementControl
탐지된 위협을 평가하고 요격 미사일 할당을 결정한다.

- **우선순위 점수 산출:**
  - 속도 (빠를수록 높은 점수)
  - 고도 (낮을수록 높은 점수 — 요격 시간 촉박)
  - 방어 대상까지 남은 거리 (가까울수록 높은 점수)
- 교전 가능 시간창(Engagement Window) 계산
- 잔여 요격 미사일 수 관리

### InterceptMissile
3단계 유도 방식으로 표적을 추적한다.

| 단계 | 유도 방식 | 동작 |
|------|----------|------|
| Phase 1 (INS) | 관성항법 | 발사 직후, 예측 위치로 자체 비행 |
| Phase 2 (지령유도) | 레이더 지령 | 레이더가 미사일과 표적 동시 추적, 보정 명령 전송 |
| Phase 3 (능동호밍) | 자체 레이더 | 미사일 내장 레이더로 표적 직접 추적 |

- 거리 기준으로 단계 자동 전환
- 각 단계별 조향(steering) 로직 분리 구현

### HitJudgement
요격 미사일과 위협 간 충돌 판정을 처리한다.

- 매 프레임 거리 계산
- 임계 거리 이하 → 요격 성공
- 위협이 방어 대상 반경 내 진입 → 피격 판정

### Metrics Logger
시뮬레이션 데이터를 수집하고 저장한다.

- 위협 종류별 요격 성공/실패 횟수
- 소모된 요격 미사일 수
- 방어 대상 피격 횟수
- CSV / JSON 저장

### Visualizer
- **pygame:** 실시간 2D 렌더링 (위협, 요격 미사일, 레이더 범위, 폭발 이펙트)
- **matplotlib:** 시뮬레이션 종료 후 메트릭 그래프 출력
